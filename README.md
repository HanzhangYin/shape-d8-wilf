# Shape-Wilf + D8 closure experiments

This small repository packages the script and input data used to generate candidate ordinary Wilf classes from:

1. shape-Wilf equivalence classes, and
2. the square-dihedral D8 action on permutation plots.

The main script is:

```bash
python3 generate_wilf_from_shape_d8.py
```

The input files `data/reverse_k*.json` use the legacy key `record_classes`; in this project those entries are interpreted as shape-Wilf classes.

## Mathematical interpretation

For a fixed length `k`:

- vertices are shape-Wilf classes from `reverse_k*.json`;
- an edge connects two vertices if some pattern in one class has a D8 image in the other;
- connected components are the classes generated by shape-Wilf equivalence plus D8 symmetry.

D8 is the square-dihedral group acting on permutation plots. The script generates it using:

- reverse,
- complement,
- inverse.

Each D8 operation preserves ordinary avoidance counts. Shape-Wilf equivalence is also Wilf-preserving. Thus each generated component is a positive certificate: all patterns inside the component are ordinary Wilf-equivalent.

Important caveat: the script does not prove that different generated components are non-Wilf-equivalent. That requires separate ordinary-count separation evidence or comparison to a trusted Wilf-class table.

## Data included

The `data/` directory contains:

- `reverse_k5.json`
- `reverse_k6.json`
- `reverse_k7.json`
- `reverse_k8.json`
- `wilf.txt`

The smaller `reverse_k3.json` and `reverse_k4.json` files are intentionally not bundled. The repository starts its bundled reverse-data examples at `k=5`.

The `reverse_k*.json` files contain shape-Wilf classes for the corresponding `S_k` datasets. Some files may omit D8 images that are singleton shape-Wilf classes. By default, the script completes the universe under D8 by adding missing D8 images as singleton shape-Wilf classes.

Known examples:

- `reverse_k5.json` has 119 patterns and the script adds `23451`.
- `reverse_k8.json` has 40319 patterns and the script adds `23456781`.

## Quick start

Generate the S5 closure and compare against the included ordinary Wilf table:

```bash
python3 generate_wilf_from_shape_d8.py \
  --input data/reverse_k5.json \
  --output-json outputs/generated_wilf_k5_from_shape_d8.json \
  --output-txt outputs/generated_wilf_k5_from_shape_d8.txt \
  --check-wilf data/wilf.txt
```

Generate the S8 closure:

```bash
python3 generate_wilf_from_shape_d8.py \
  --input data/reverse_k8.json \
  --output-json outputs/generated_wilf_k8_from_shape_d8.json \
  --output-txt outputs/generated_wilf_k8_from_shape_d8.txt
```

Expected S8 summary:

```text
input shape-Wilf classes: 39326
input patterns: 40319
completed patterns: 40320
D8-added singleton patterns: 23456781
generated classes: 4755
generated non-singleton classes: 4755
size distribution: {2: 29, 4: 260, 8: 4094, 12: 10, 16: 309, 18: 1, 20: 1, 24: 39, 32: 9, 40: 1, 48: 1, 56: 1}
```

## Reproduce all bundled examples

```bash
mkdir -p outputs
for k in 5 6 7 8; do
  python3 generate_wilf_from_shape_d8.py \
    --input data/reverse_k${k}.json \
    --output-json outputs/generated_wilf_k${k}_from_shape_d8.json \
    --output-txt outputs/generated_wilf_k${k}_from_shape_d8.txt
done
```

For the lengths present in `data/wilf.txt`, you can also run checks, for example:

```bash
python3 generate_wilf_from_shape_d8.py \
  --input data/reverse_k5.json \
  --output-json outputs/generated_wilf_k5_from_shape_d8.json \
  --output-txt outputs/generated_wilf_k5_from_shape_d8.txt \
  --check-wilf data/wilf.txt
```

## Development

No third-party Python packages are required. Python 3.10+ is recommended.

Run tests:

```bash
python3 -m unittest discover -s tests
```

## Status

This is an experiment/reproducibility package. Generated components should be described as classes generated by shape-Wilf equivalence plus D8 closure, not as a general theorem about all Wilf classes unless independently verified for the dataset in question.
